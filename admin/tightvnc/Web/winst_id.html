Instalasi

TightVNC tersedia sebagai paket yang dapat dieksekusi dan terinstal sendiri. Jalankan saja, dan Anda akan melihat sebuah wizard instalasi yang sudah lazim yang akan mengizinkan Anda untuk membaca catatan instalasi, pilih direktori tujuan, set opsi instalasi dan seterusnya.

Stelah program instalasi selesai, bagian "TightVNC" baru akan muncul di menu Start->Program. Jika Anda ingin memulai TightVNC secara otmooatis dan menjalankannya bahkan saat ada user yang masuk log, Anda dianjurkan untuk menginstalasinya sebagai servis sistem dengan menggunakan item menu yang bersangkutan dari bagian Administrasi TightVNC. Mohon dicatat bahwa instalasi servis WinVNC adalah satu-satunya cara untuk membuat Ctrl-Alt- Del untuk bekerja jarak jauh (teraplikasikan hanya pada sistem Windows NT/2000/XP).

TightVNC secara bawaan diinstall ke direktori Program Files\TightVNC, tapi Anda dapat memilih lokasi lain saat instalasi. Perlu dicatat bahwa TightVNC tidak menginstal apa pun di dalam direktori sistem jadi alternatif lain untuk menginstal TightVNC adalah dengan menyalin berkas program dan DLL ke direktori yang Anda inginkan (tapi untuk kasus ini Anda harus membuat jalan pintas ke program secara manual).

Jika Anda ingin menginstal TightVNC pada beberapa komputer, dan tidak ingin memasukkan kata sandi yang sama pada setiap mesin, instal TightVNC sekali dan set kata sandi, lalu salin pengaturan registry ke komputer lainnya. Pengaturan TightVNC dapat ditemukan pada bagian berikut ini di registry:

  • HKEY_LOCAL_MACHINE\Software\ORL\WinVNC3
  • HKEY_CURRENT_USER\Software\ORL\WinVNC3

Penginstal TightVNC dapat bekerja pada modus senyap dan tidak bertanya pada saat instalasi (tapi ia mungkin akan menunjukkan kotak kesalahan jika terjadi kesalahan). Pada modus ini TightVNC akan diinstalasi ke direktori bawaan, yaitu subdirektori TightVNC dalam Program Files). Ini adalah baris perintah untuk menginstal TightVNC dalam modus senyap: 

tightvnc-1.3.10-setup.exe /sp- /verysilent

Pembaharuan Jarak Jauh

Server TightVNC dapat diperbaharui dari jarak jauh. Ini berarti Anda dapat melakukan instalasi TightVNC dengan bekerja pada sesi TightVNC yang aktif. Pada saat servis TightVNC sedang berjalan, ianya tidak mungkin untuk menggantikan file eksesusi yang telah ada, jadi penginstal akan menyalin berkas baru ke lokasi sementara, dan berkas baru ini akan menggantikan versi lama pada reboot yang berikutnya. Penginstal akan menanyakan reboot jika ia tidak dapat menggantikan berkas program.

Sebelum menggunakan fitur ini, Anda dianjurkan untuk membaca dan mengerti hal di bawah ini:

  • Pembaharuan jarak jauh membutuhkan reboot dari komputer tersebut. Jika Anda ingin dapat mengakses komputer setelah reboot, pastikan Anda menjalankan WinVNC sebagai servis, bukan dalam modus aplikasi.
  • Jika Anda menjalankan servis WinVNC dari distribusi VNC lainnya, prosedur instalasi ini tidak akan menggantikannya dengan servis TightVNC secara bawaan. Ini adalah kelemahan yang mungkin akan diatasi dalam versi berikutnya. Namun, ada satu cara untuk menlakukan pembaharuan untuk ini: instal TightVNC pada direktori yang sama di mana berkas yang lama (WinVNC.exe dan VNCHooks.dl) diinstal. Dalam kasus ini, binari yang lama akan digantikan dengan yang baru pada saat reboot, dan tidak perlu menginstal ulang servis. Mohon dicatat bahwa penginstal seharusnya menampilkan pertanyaan reboot pada akhir dari proses instalasi, jika tidak mungkin Anda telah memilih direktori instalasi yang salah.
  • Ada beberapa hal yang mencegah mesin untuk direboot secara benar, dan ini dapat menyebabkan hilangnya kontrol atas komputer. Dengan kata lain, tidak ada GARANSI pembaharuan jarak jauh sepenuhnya dapat diandalkan. Untuk meminimalkan risiko yang mungkin, tutup semua aplikasi yang berjalan (di samping servis WinVNC) sebelum manjalankan penginstal TightVNC.

Memulai

TightVNC, tidak seperti VNC yang biasa, mempunyai dua bagian: Server (juga direfernsi sebagai WinVNC), yang membagi layar dari mesin di mana ia berjalan, dan Pemantau yang menampilkan layar jarak jauh dari server. Untuk memulai, And aperlu menjalankan sebuah server pada mesin yang Anda ingin akses secara jarak jauh, dan menhubungkannya dengan sebuah pemantau. Distribusi TightVNC untuk Windows mengikutsertakan bagian server dan pemantau.

Menjalankan Sebuah Server (WinVNC)

Server TightVNC dapat dimulai dengan satu dari dua cara berikut:

  • sebagai aplikasi (untuk pemakai saat ini saja);
  • sebagai servis Windows (operasi menyeluruh sistem).

Dalam modus aplikasi, server dapat dijalankan hanya pada sesi pemakai saat ini, dan akan ditutup pada saat keluar log. Untuk memulai WinVNC dalam modus aplikasi, pilih Start->Programs->TightVNC->Launch TightVNC Server.

Agar mesin dapat diakses pada saat tidak ada pengguna yang masuk log, dan agar server dimulai secara otomatis saat reboot, Server TightVNC seharusnya dijalankan sebagai servis sistem. Untuk menginstalasinya sebagai servis, pilih Start->Programs->TightVNC->Administration->Install VNC Service. Dalam Windows 95/98/ME ia akan memulai servis secara langsung, sedangkan pada Windows NT/2000/XP Anda harus memulai servis secara manual dari Control Panel atau dari perintah baris (net start winnvc). Dalam kasus manapun, ianya akan memulai secara otomatis pada reboot sistem selanjutnya. Perlu dicatat bahwa jike servis VNC dimulai secara manual, Anda mungkin harus menjalankan Start->Programs->TightVNC->Administration->Run Service Helper, untuk membuat WinVNC menampilkan ikon baki (lihat di bawah).

Tidak tergantung dari bagaimana Server TightVNC dimulai, ia akan menampilkan jendela Properties pada startup pertama, untuk mengizinkan Anda untuk memasukkan kata sandi yang akan Anda pergunakan dalam pemantau untuk mengakses mesin. Sampai kata sandi dimasukkan, TightVNC tidak akan menerima koneksi jaringan yang masuk.

Dalam konfigurasi bawaan, setiap pemakai dapat memiliki kata sandi WinVNC yang terpisah, ada juga kata sandi bawaan yang khusus yang dipergunakan jika tidak ada kata sandi yang tersedeia (contoh: pada saat tidak ada yang masuk log, atau jika tidak ada kata sandi pemakai yang diatur). Catatan: tidak ada kata sandi bawaan yang telah didefinisikan di dalam TightVNC, kata sandi mesin dinamai "bawaan" hanya karena ia dipergunakan ketika tidak ada kata sandi spesifik untuk pemakai yang tersedia.

Untuk mengatur kata sandi pemakai, akses ke dialog Properties dari menu, seperti dijelaskan di bawah ini. Jika Anda ingin mengatur kata sandi bawaan, jalankan Start->Programs->TightVNC->Administration->Show Default Settings. Catatan: Judul dari dialog Properties menampilkan ":Current User Properties" atau "Default Local System Properties", tergantung kelompok pengaturan mana yang disunting.

Pada startup yang sukses, TightVNC akan menambahkan sebuah ikon kecil pada baki dalam task bar sistem (area pemberitahuan). Ikon itu memiliki latar belakang putih jika tidak ada pemantau yang terkoneksi, dan warna berlawanan jika setidaknya ada satu pemantau yang mengakses mesin. Menggerakkan tetikus di atas ikon ini menunjukkan alamat IP dari mesin (yang dapat dimasukkan dalam pemantau jarak jauh untuk mengakses server).

Klik kanan pada ikon baki untuk menunjukkan menu:


Menu dari Server TightVNC

Perintah menu berikut ini tersedia:

  • Properties... - Ini akan mengakibatkan dialog Properties ditampilkan, mengizinkan pemakai untuk mengubah berbagai parameter dari Server TightVNC. Dialog ini akan ditampilkan juga pada saat klik-ganda pada ikon baki.
  • Add New Client... - Ini mengizinkan koneksi keluar untuk dibuat dari server ke pemantau dimulai pada saat modus "dengar". Nama dari mesin pemantau target dan nomor tampilan (opsional) dapat dimasukkan ke dalam dialog. Koneksi diciptakan dengan cara ini dianggap terbagi. "Koneksi kembali" dapat juga dimulai dari perintah baris dengan menggunakan opsi -connect.
  • Kill All Clients - Ini akan memutuskan semua klien yang terkoneksi dari server.
  • Disable New Clients - Ini akan mematikan koneksi klien baru secara sementara ke server. Pilih item menu yang sama untuk mengaktifkan kembali koneksi klien baru. Catat bahwa modus ini tidak dikembalikan pada saat server dimulai kembali.
  • About... - Menunjukkan informasi tentang perangkat lunak.
  • Close TightVNC Server - Keluar.

Menjalankan Sebuah Pemantau

Untuk memantau dan mengontrol mesin jarak jauh di mana Server TightVNC dijalankan, Anda perlu menjalankan Pemantau TightVNC. Untuk menjalankan pemantau, pilih Start->Programs->TightVNC->TightVNC Viewer. Anda akan melihat sebuah jendela yang mengizinkan Anda untuk memilih server tujuan.


Jendela dialog "Koneksi Baru"

Setelah memasukkan nama hosti dan alamat IP-nya (dan secara opsional nomor tampilannya), pilih "Connect". Pada koneksi yang berhasil, Anda akan ditanyakan kata sandi, dan jika kata sandi yang dimasukkan benar, akhirnya Anda bisa melihat mesin jarak jauh.

Secara alternatif, Anda dapat memulai pemantau dalam modus dengar, dengan menggunakan tombol yang bersangkutan di dalam jendela "New Connection". Dalam modus itu, ikon pemantau akan tampil di dalam baki sistem, dan ia akan menerima koneksi terbalk dari server TightVNC (lihat deskripsi di atas tentang item menu WinVNC "Add New Client".

Dalam jendela "New Connection", Anda dapat menggunakan konteks bantuan yang ada. Untuk mendapatkan bantuan dengan menggunakan kontrol yang ada, pertama klik sebuah tombol tanda tanya pada judul jendela, lalu klik pada kontrol tersebut. Cara lain untuk mendapatkan bantuan konteks adalah dengan menekan tombol F1 pada saat fokus papan tombol sedang berada pada kontrol yang bersangkutan.

Menggunakan Sebuah Peramban Web Sebagai Pemantau

Server VNC juga dapat berisi server Web kecil. Jika Anda melakukan koneksi kepadanya dengan peramban web, versi Java dari pemantau akan diunduh secara otomatis, yang mengizinkan Anda untuk mengakses desktop jarak jauh. Jelasnya, peramban Web harus mendukung applet Java. Anda juga dianjurkan tidak memakai proxy, untuk mengizinkan aplet Java mengakses server jarak jauh secara langsung.

Server mendengar koneksi HTTP pada port 5800 + nomor tampilan. (Ingat mesin WinVNC secara bawaan adalah tampilan 0.) Jadi untuk melakukan koneksi ke tampilan 2 pada mesin "myhost", Anda ingin menunjukkan peramban web pada http://myhost:5802/ . Applet akan menanyakan kata sandi Anda, dan selanjutnya akan menampilkan desktop.

Membongkar TightVNC

TightVNC dapat dibongkar menggunakan Add/Remove Programs di dalam Control Panel, tapi Anda juga dapat menghilangkan direktori yang Anda instal (biasanya C:\Program Files\TightVNC). Perlu dicatat bahwa instalasi TightVNC tidak menyalin berkas apapun ke direktori sistem. Sebelum membongkar TightVNC, pastikan bahwa WinVNC tidak berjalan dant idak diinstalasi sebagai servis.
